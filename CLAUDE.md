## 交流风格

- 用中文交流
- 直接给结论，不要过度解释
- 有问题直接问，不要猜测意图
- 报告结果：成功说"完成了"，失败给具体错误

## 代码偏好

- 保持简洁，不过度设计
- 匹配现有代码风格
- 不擅自添加没要求的功能
- 写可读的代码，不写炫技代码

## Git 规则

- 不要擅自 commit 或 push，除非明确要求
- Commit message 用英文

## 文件操作

- 创建新文件用 Write，修改已有文件用 Edit
- 优先用 Glob/Grep 搜索，不用 shell 命令

## 智能模型与花费管理

用户已授权以下自动化权限：

**任务难度判断**：
- 日常增删改查、简单修 Bug、搜索文件 → 用 Sonnet（默认，已配置）
- 多文件架构设计、棘手 Bug 调试、深度重构 → 主动建议 `/model opus`
- 用户对回答不满意 → 主动建议切 Opus 重试
- 不用等用户问，主动判断并建议

**花费意识**：
- 阶段性任务完成后自我评估 Token 消耗
- 单次任务消耗异常高时提醒用户
- 会话超过 15 轮主动建议 `/compact`

**输出精简**：
- 简单任务直接给结论
- 复杂任务才详细展开
- 完成了就说"完成了"，不重复描述
